# Анализ проекта ChineseWeb

## Структура проекта
- **Бэкенд**: Node.js + Express + SQLite (Sequelize ORM)
- **Фронтенд**: Vanilla JS (ES6), без фреймворков
- **Данные**: словарь БКРС (176K+ записей), HSK 3.0 (872KB JSON)

## План исправления

### Фаза 1 - Критические ошибки
1. Получить/создать `bkrs_complete.json` (запустить парсер v5 на исходном HTML)
2. Исправить кодировку HSK данных - перепарсить PDF или найти чистый источник
3. Исправить Docker healthcheck: `CMD node -e "require('http').get(...)"`
4. Создать `.env` из `.env.example`
5. Удалить мёртвый `seed.js`

### Фаза 2 - Основные исправления
6. Исправить алгоритм SRS (правильный SM-2)
7. Заменить random endpoint на `ORDER BY RANDOM() LIMIT 1` (SQLite)
8. Добавить валидацию входных данных
9. Заменить `prompt()` на модальное окно в frontend
10. Добавить поле pinyin в модель Dictionary и seed-данные

### Фаза 3 - Качество
11. Добавить интеграционные тесты (supertest)
12. Добавить rate limiting (express-rate-limit)
13. Добавить миграции (umzug или sequelize-cli)
14. Добавить индикаторы загрузки и обработку ошибок во frontend
