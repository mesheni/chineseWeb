## 🔴 КРИТИЧЕСКИЕ БАГИ (ломают функциональность)

### 1. Study mode: карточка не переворачивается при «Показать ответ»
**Файл:** `public/js/app.js:391-395`
**Описание:** При клике на «Показать ответ» в режиме изучения только прячется кнопка и показываются «Знаю»/«Не знаю». Флип карточки (`.flipped`) НЕ происходит. Перевод остаётся невидимым на задней стороне карточки — пользователь никогда не видит перевод.
**Исправление:** Добавить `document.getElementById('card').classList.add('flipped')` в обработчик `studyShowAnswer`.

### 2. Study mode: перевод показан ДО нажатия «Показать ответ»
**Файл:** `public/js/app.js:377`
**Описание:** `e.studyTranslation.textContent = word.russian_word` заполняет перевод сразу при показе слова, хотя визуально он скрыт за карточкой. После исправления бага №1 перевод станет виден. Логика правильная, но сейчас перевод заполняется на невидимой стороне — это ок, если флип заработает.

### 3. Study mode и Review mode: авто-озвучка без согласия пользователя
**Файл:** `public/js/app.js:388-389, 467`
**Описание:** Комментарий гласит «Speak only if user previously clicked speak (opt-in)», но код вызывает `speakChinese()` всегда при показе каждого слова. Это раздражает, особенно при быстром просмотре. В Review режиме тоже самое.
**Исправление:** Добавить флаг `speakAutoEnabled`, включаемый по клику на кнопку 🔊, или сделать галку «Авто-озвучка».

### 4. Review mode: «Начать заново» не перезапрашивает слова
**Файл:** `public/js/app.js:494-495`
**Описание:** `reviewRestart` просто сбрасывает `reviewIndex = 0` и показывает те же уже пройденные слова. Если все due-слова закончились, рестарт показывает Done. Не перезапрашивает `GET /review` с сервера.
**Исправление:** При рестарте заново вызывать `GET /:id/review` для свежей выборки due-слов.

### 5. Test mode: неправильно ищет `list_id` для SRS-отправки
**Файл:** `public/js/app.js:566-583`
**Описание:** В `handleTestAnswer` используется `found.list_id`, но `found` ищется по `entry.id` в `testQueue`. Переменная `testListId` уже хранит правильный ID списка, но не используется. Может привести к ошибке если структура `testQueue` не содержит `list_id`.
**Исправление:** Использовать `testListId` вместо `found.list_id`.

## 🟡 БАГИ ВЫСОКОГО ПРИОРИТЕТА

### 6. Study mode: кнопки «Знаю»/«Не знаю» не отправляют данные
**Файл:** `public/js/app.js:397-398`
**Описание:** Нажатие «Знаю»/«Не знаю» просто переходит к следующему слову. Никакие SRS-данные не отправляются на сервер. Режим «Учить» полностью оторван от SRS.
**Исправление:** Добавить отправку `POST /:listId/review` с quality=4 (знаю) или quality=1 (не знаю).

### 7. Dictionary: поиск не ищет по пиньинь
**Файл:** `server/routes/dictionary.js:24-28`
**Описание:** `Op.or` включает только `chinese` и `russian_word`. Пользователь не может найти слово по пиньинь (например, «nihao» → 你好).
**Исправление:** Добавить `{ pinyin: { [Op.like]: `%${q}%` } }` в `Op.or`.

### 8. `MAX_NEW_WORDS_PER_DAY` не используется
**Файл:** `.env.example:3`, все файлы
**Описание:** Переменная окружения объявлена, но нигде в коде не проверяется. Нет ограничения на добавление новых слов в день.
**Исправление:** Добавить проверку в `POST /:id/words` — считать слова, добавленные сегодня, и отклонять при превышении лимита. Или убрать переменную из конфига.

### 9. Неправильная работа фильтра длины иероглифов в поиске
**Файл:** `public/js/app.js:95-101`, `public/index.html:25`
**Описание:** В `doDictSearch()` фильтр длины применяется всегда, даже когда query пустой. Но если query пуст, функция выходит раньше (строка 97). При этом дефолтное значение фильтра — «2 иероглифа» (`selected`). Это сбивает с толку: пользователь ищет пустой запрос, видит hint, но не понимает, что фильтр игнорируется.
**Исправление:** Позволить поиск без query с фильтром длины (показать все слова заданной длины). Либо скрывать фильтр при пустом запросе.

### 10. Поиск по словарю: SQL LIKE инъекция символов `%` и `_`
**Файл:** `server/routes/dictionary.js:25-27`
**Описание:** Пользовательский ввод напрямую подставляется в `[Op.like]`. Символ `%` в запросе работает как wildcard, `_` как любой один символ. Это не дыра безопасности, но даёт неожиданные результаты.
**Исправление:** Экранировать `%`, `_`, `\` в поисковом запросе перед LIKE.

### 11. Ошибки сервера раскрывают внутренние детали
**Файл:** `server/routes/dictionary.js:43`, `server/routes/studyLists.js:36,49,62,82,114,126,150,184,204,226,266`
**Описание:** `error.message` возвращается клиенту. Может раскрыть структуру БД, пути файлов и т.д.
**Исправление:** В production окружении возвращать `{ error: 'Внутренняя ошибка сервера' }`, а детали писать в `console.error`.

## 🟠 БАГИ СРЕДНЕГО ПРИОРИТЕТА

### 12. Review mode: кнопки качества (1-5) без качества 2
**Файл:** `public/index.html:124-127`
**Описание:** Кнопки: «Забыл»=1, «Тяжело»=3, «Нормально»=4, «Легко»=5. Качество 2 пропущено. По SM-2 качество 0-2 это провал (сброс интервала), 3-5 — успех. Отсутствие 2 не критично, но неполно.
**Исправление:** Добавить кнопку «Почти» (quality=2) или оставить как есть — UX-решение.

### 13. Study mode: пиньинь всегда виден на лицевой стороне карточки
**Файл:** `public/index.html:75`, `public/js/app.js:376`
**Описание:** Пиньинь показывается сразу вместе с иероглифом. Это лишает смысла проверку «знаю ли я чтение». Пользователь видит и иероглиф, и произношение одновременно.
**Исправление:** Скрывать пиньинь до флипа карточки (после нажатия «Показать ответ»). Добавить переключатель «Показывать пиньинь».

### 14. HSK import: нет индикатора прогресса
**Файл:** `server/routes/studyLists.js:231-268`, `public/js/app.js:310-329`
**Описание:** Импорт HSK 6 (1140 слов) в цикле по одному слову — медленно, нет прогресс-бара. Кнопка просто меняется на «⏳ Импорт...» на всё время.
**Исправление:** Использовать `bulkCreate` вместо цикла `for` на сервере. Добавить прогресс на фронте.

### 15. Dictionary: `definition` всегда пустой для HSK-данных
**Файл:** `public/js/app.js:134`
**Описание:** `escHtml(sample.definition || '').slice(0, 200)` — HSK JSON не содержит поля definition, только word/pinyin/translation. Отображается пустая строка.
**Исправление:** Не показывать блок `.dict-definition` если definition пуст. Или добавить примеры употребления в HSK JSON.

### 16. Test mode: опции-дистракторы слишком простые
**Файл:** `public/js/app.js:544-549`
**Описание:** Неправильные варианты выбираются случайным образом из всех слов списка. Варианты могут быть совершенно непохожими на правильный, делая тест слишком лёгким.
**Исправление:** Выбирать дистракторы из слов того же HSK уровня или семантически близких (по длине, по первой букве перевода).

### 17. CSS: нет активного состояния для кнопок навигации на мобильных
**Файл:** `public/css/style.css:126-155`
**Описание:** `.nav-tab:hover` работает только на десктопе. На мобильных нет `:active` стиля для нажатия на таб.
**Исправление:** Добавить `.nav-tab:active { background: var(--bg-card); }`.

## 🟢 УЛУЧШЕНИЯ UI/UX

### 18. Дизайн: нет переключателя тёмная/светлая тема
**Описание:** Всегда тёмная тема. Нет accessibility для пользователей, предпочитающих светлый интерфейс.
**Предложение:** Добавить CSS-переменные для светлой темы, кнопку переключения в header, сохранять выбор в localStorage.

### 19. Дизайн: нет клавиатурных сокращений
**Описание:** Нельзя использовать Enter для флипа карточки, цифры 1-5 для качества в review, стрелки для навигации.
**Предложение:** Добавить глобальный обработчик `keydown`:
- `Space`/`Enter` — показать ответ / флип
- `1-5` — выбор качества в review
- `←` `→` — навигация по словам в study
- `R` — повторить произношение

### 20. Дизайн: иконки навигации — emoji, а не SVG
**Описание:** Emoji выглядят по-разному на разных ОС. На Windows некоторые emoji чёрно-белые.
**Предложение:** Заменить на SVG-иконки (inline или icon font) для консистентного отображения.

### 21. Дизайн: нет favicon
**Описание:** Вкладка браузера показывает пустую иконку. Не хватает `/favicon.ico`.
**Предложение:** Добавить простой favicon с иероглифом 汉 или 中.

### 22. Дизайн: карточка в тестовом режиме использует тот же `#test-card`, но без флипа
**Описание:** В test mode карточка показывает иероглиф + пиньинь, но это не flashcard — это просто отображение. Структура внутри `.flashcard > .card` с `.card-front` избыточна.
**Предложение:** Упростить HTML для test mode: убрать `.card-back`, использовать простое отображение.

### 23. UX: нет подтверждения при выходе из review/study/test
**Описание:** При переключении вкладки во время сессии обучения прогресс теряется без предупреждения.
**Предложение:** Показывать `confirm()` или сохранять состояние при переключении вкладок.

### 24. UX: статистика слишком базовая
**Описание:** Только 3 числа: всего слов, изучено, на сегодня. Нет графиков, нет истории, нет streak-счётчика.
**Предложение:**
- Добавить график прогресса по дням (Chart.js или canvas)
- Счётчик «дней подряд» (streak)
- Распределение по ease-factor (лёгкие/сложные слова)
- Круговая диаграмма: новые / изученные / проблемные

### 25. UX: нет сортировки слов в просмотре списка
**Описание:** `GET /:id/words` возвращает слова в порядке `id DESC`. Нет возможности сортировать по дате добавления, по алфавиту, по следующему повторению.
**Предложение:** Добавить query-параметры `sort` и `order` в API, добавить сортировку в UI.

### 26. UX: словарь не показывает уровень HSK
**Описание:** В результатах поиска нет информации об уровне HSK. Пользователь не знает, к какому уровню относится слово.
**Предложение:** Добавить бейдж «HSK 3» в `.dict-entry`.

### 27. UX: повторение — нет счётчика оставшихся слов в реальном времени
**Описание:** `reviewPosition` показывает «Слово X из Y», но Y — это изначальное количество due-слов. После сабмита review слово удаляется из due, но счётчик не обновляется.
**Исправление:** Обновлять `reviewCount` после каждого ответа.

### 28. UX: нет поиска по списку слов внутри режима просмотра списка
**Описание:** При просмотре списка из 1000+ слов нет поиска/фильтрации внутри списка.
**Предложение:** Добавить строку поиска в `listDetail`.

### 29. UX: нет массовых операций со списком
**Описание:** Нельзя выбрать несколько слов для удаления, перемещения или изменения приоритета.
**Предложение:** Добавить чекбоксы и кнопки «Удалить выбранные», «Сбросить SRS».

### 30. UX: импорт HSK на странице списков не показывает частичный прогресс
**Описание:** Карточка HSK показывает только «Импортирован» или кнопку импорта. Не показывает количество уже добавленных слов из этого уровня.
**Предложение:** Показывать «X/Y слов в списке» для каждого уровня HSK.

## 📋 НЕДОСТАЮЩИЕ ФУНКЦИИ ДЛЯ ПОЛНОТЫ

### 31. Нет секции грамматики
**Описание:** В ROADMAP заявлена грамматика (20-30 правил, простые упражнения). Полностью отсутствует.
**Предложение:** Добавить:
- Таблицу `grammar_rules` (id, title, explanation, examples, level)
- Таблицу `grammar_exercises` (id, rule_id, sentence, options, correct)
- Страницу «Грамматика» с правилами и тестами

### 32. Нет примеров употребления слов
**Описание:** HSK JSON содержит только word/pinyin/translation. Нет примеров предложений с переводом.
**Предложение:** Дополнить HSK JSON полем `examples: [{chinese, russian}]`. Либо загрузить из отдельного источника. Добавить отображение примеров на карточке и в словаре.

### 33. Нет практики письма (stroke order)
**Описание:** Приложение для изучения китайского без отработки написания иероглифов неполноценно.
**Предложение:**
- Добавить canvas для рисования иероглифов
- Использовать HanziWriter (JS-библиотека) для показа порядка черт
- Режим «Письмо»: показать перевод → пользователь рисует иероглиф → проверка

### 34. Нет аудио-файлов (только TTS)
**Описание:** Web Speech API не работает офлайн, качество зависит от ОС/браузера. Нет fallback.
**Предложение:** Добавить возможность загрузки аудиофайлов для слов, либо использовать бесплатное API (например, Google TTS или Forvo).

### 35. Нет listening comprehension (аудирование)
**Описание:** Нет режима, где произносится слово, а пользователь выбирает перевод/иероглиф.
**Предложение:** Добавить режим «Аудирование»: проигрывается звук → выбор из 4 вариантов (иероглиф или перевод).

### 36. Нет экспорта/импорта списков
**Описание:** Нельзя выгрузить список в JSON/CSV для бэкапа или поделиться.
**Предложение:** Кнопки «Экспорт JSON» / «Импорт JSON» для study lists.

### 37. Нет ежедневной цели и streak
**Описание:** Нет мотивационного механизма. Нет счётчика «дней подряд», нет цели «X слов в день».
**Предложение:** Добавить в статистику:
- Streak (дни подряд с хотя бы одним повторением)
- Дневная цель (настраиваемая)
- Прогресс-бар выполнения цели

### 38. Нет очереди новых слов (new words queue)
**Описание:** `MAX_NEW_WORDS_PER_DAY` не реализован. Нет контроля темпа изучения новых слов.
**Предложение:** При добавлении слов в список учитывать дневной лимит. Показывать «Сегодня можно добавить ещё X слов».

### 39. Нет фильтрации слов по сложности / известности
**Описание:** Нельзя отфильтровать список по «проблемным» словам (низкий ease factor, много неправильных ответов).
**Предложение:** Добавить теги/категории: «Проблемные», «Новые», «Уверенно знаю». Фильтры в просмотре списка.

### 40. Нет OCR / поиска по изображению
**Описание:** Нельзя сфотографировать иероглиф и найти его в словаре.
**Предложение:** (Опционально, сложно) Интеграция с OCR API для распознавания иероглифов с камеры.

### 41. Нет тёмной/светлой темы (дублирует №18)
**См. пункт 18 выше.**

## 🔧 ТЕХНИЧЕСКИЙ ДОЛГ

### 42. `console.log` в production коде
**Файлы:** `server/seed-hsk.js`, `server/app.js`
**Предложение:** Заменить на logging framework (winston/pino) с уровнями.

### 43. Нет CI/CD
**Описание:** Нет GitHub Actions / GitLab CI для авто-тестов и сборки Docker.
**Предложение:** Добавить `.github/workflows/test.yml` с `npm test` и `docker build`.

### 44. SQLite в production
**Описание:** SQLite не подходит для конкурентных записей. При росте пользователей нужна миграция на PostgreSQL.
**Предложение:** Документировать это ограничение. Для MVP ок.

### 45. Нет миграций БД
**Описание:** `sequelize.sync()` не версионирует схему. При изменении модели возможна потеря данных.
**Предложение:** Добавить `sequelize-cli` и миграции.

### 46. Frontend: дублирование id в HTML
**Файл:** `public/index.html`
**Описание:** ID `card`, `flashcard`, `card-front`, `card-back` не имеют префикса режима, в отличие от `reviewCard`, `review-flashcard` и т.д. Это работает (разные DOM-контексты), но неконсистентно.
**Предложение:** Переименовать `#card` → `#studyCard`, `#flashcard` → `#studyFlashcard`.

### 47. Тесты используют ручной test runner вместо фреймворка
**Файл:** `tests/api.test.js`
**Описание:** Самописный `test()` без `before/after`, без таймаутов, без отчётов.
**Предложение:** Мигрировать на Mocha/Jest с `supertest`.
